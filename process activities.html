Process Activities

Real software processes are interleaved sequences of technical, collaborative, and managerial activities aimed at specifying, designing, implementing, and testing software systems. Software developers use various tools to support editing documents and managing the large amount of information generated in projects.

Overview of Software Process Activities
The four basic process activities — specification, development, validation, and evolution — are organized differently across software development methods. In the waterfall model, these activities occur sequentially, while in incremental development they are interleaved to allow flexibility and faster feedback.

2.2.1 Software Specification

Software specification or requirements engineering defines system services and constraints. It aims to produce an agreed document satisfying stakeholder needs.

Main Activities in Requirements Engineering

1

Feasibility Study

Assesses if requirements are realistic and achievable within constraints.

2

Elicitation & Analysis

Gathers and analyzes stakeholder requirements and priorities.

3

Specification

Documents the user and system requirements clearly and precisely.

4

Validation

Ensures requirements are realistic, consistent, and complete before development.

2.2.2 Software Design and Implementation

Software design transforms specifications into an executable system. Agile methods often combine design and implementation, emphasizing working software over documentation.

Key Design Activities

1. Architectural design — defines overall structure and components.
2. Interface design — defines connections between components.
3. Component design — defines internal structures of components.
4. Database design — defines data structures and storage design.

2.2.3 Software Validation

Validation ensures that the software meets requirements and user expectations. Testing is the main validation activity.

Testing Stages

1. Development testing — tests individual components.
2. System testing — tests integrated components as a whole.
3. Acceptance testing — performed by customers before deployment.

2.2.4 Software Evolution

Software evolves as requirements, technologies, and user needs change. Maintenance and evolution ensure that software remains reliable and relevant, adapting to new business goals, user feedback, and technological advances. Continuous updates, refactoring, and feature enhancements help software stay effective and sustainable over time.

Key Takeaway:

Software evolution ensures that systems can adapt, improve, and remain valuable as environments and user expectations change.
