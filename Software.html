Process Activities
“Real software processes are interleaved sequences of technical, collaborative, and managerial activities aimed at specifying, designing, implementing, and testing software systems.”
2.2.1 Software Specification
“Software specification defines system services and constraints. It involves understanding user needs and translating them into clear, testable requirements.”
1. Feasibility Study
Evaluates whether the proposed system can be built within technical and financial limits.
2. Requirements Elicitation & Analysis
Involves gathering, understanding, and analyzing stakeholder needs and system requirements.
3. Requirements Specification
Converts gathered requirements into an organized, documented specification for developers.
4. Requirements Validation
Ensures the requirements are realistic, complete, consistent, and meet user expectations.
2.2.2 Software Design and Implementation
“Design and implementation transform specifications into an executable system. These processes often overlap and interact frequently.”
1. Architectural Design
Defines the overall structure of the system and identifies the major components.
2. Interface Design
Specifies how system components interact and communicate with each other.
3. Component Design
Defines the internal structure and behavior of individual software components.
4. Database Design
Specifies data organization, relationships, and storage strategies.
“Agile methods often combine design and implementation, emphasizing working software over detailed design documents.”
2.2.3 Software Validation
“Validation ensures that the software meets requirements and expectations. Testing is the main validation activity.”
1
Development testing — tests individual components.
2
System testing — tests integrated components as a whole.
3
Acceptance testing — performed by customers before deployment.
2.2.4 Software Evolution
“Software evolves as requirements, technologies, and user needs change. Maintenance and evolution are vital to ensuring the software remains relevant.”
